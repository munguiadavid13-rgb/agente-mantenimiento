Agente Buchholz

Mantenimiento predictivo de subestaciones eléctricas

Un agente que lee las mediciones de las visitas de mantenimiento, las clasifica, sigue su tendencia en el tiempo y avisa de fallas incipientes.

🛡️ Homenaje al relé Buchholz — aviso temprano de fallas en transformadores

Jose David Pérez Munguía · Maestría en Automatización Industrial · UTH 2026.4

El problema

El mantenimiento de interruptores y restauradores genera muchas mediciones en cada visita, pero:

  • Los datos quedan dispersos en hojas sueltas por visita.
  • Cuesta ver la tendencia de un equipo a lo largo del tiempo.
  • Las fallas incipientes (tierra que sube, batería que baja…) se detectan tarde.
  • No hay un aviso automático cuando algo se sale de norma.

La idea clave

El histórico de cada parámetro (tierra, voltajes, corrientes…) es el corazón predictivo: comparar cómo evoluciona permite anticipar el problema antes de la falla.

La solución: ¿qué hace?

1
Lee el Excel de visitas
2
Clasifica cada medición
3
Sigue el histórico/tendencia
4
Notifica por 3 canales

Cada medición se etiqueta como NORMAL REVISAR MONITOREO según su criterio.

El razonamiento lo hace la sesión de Claude Code; las tools en Python hacen lo determinista (leer, clasificar, graficar, enviar). Sin costo por token de una API externa.

Arquitectura: 4 clases en composición

Subestación  Ceiba Térmica, San Isidro, …
Equipo  interruptor / restaurador + ficha técnica
Visita  fecha · cuadrilla · observación
Medición  parámetro · valor · clasificar()

Diseño elegante

  • El estado no se guarda: se calcula al vuelo comparando el valor contra su límite.
  • Si cambian los límites, se reevalúa todo el histórico.
  • Equipo.historico(p) y Equipo.tendencia(p) dan la base predictiva.

Lógica de clasificación

Grupo A — con límite → se clasifica

  • Voltaje VDC  REVISAR si ≤ 20 V
  • Voltaje AC  REVISAR si ≤ 119 V
  • Resistencia de tierra  REVISAR si ≥ 5 Ω

Grupo B — sin límite → solo histórico

  • Corriente Fase A/B/C, Potencia activa/reactiva
  • Corriente de fuga, Nº de operaciones
  • Se marcan MONITOREO y alimentan la tendencia

Regla: si el parámetro está en la tabla de Límites → se clasifica; si no → es solo monitoreo. Se cambia el comportamiento editando la tabla, sin tocar código.

La base de datos (Excel)

Un libro con:

  • Hoja índice Equipos (subestación, código, tipo)
  • Una hoja por equipo: ficha técnica + tabla de visitas
  • Hoja Límites (los criterios del Grupo A)

Agregar un equipo = fila en Equipos + su hoja (lo hace agregar_equipo.py).

Tres canales de salida + automatización

📧 Correo (HTML)

Una sola notificación con las alertas enumeradas (1, 2, 3…), con motivo, última visita y enlace al dashboard.

💬 Telegram

Mensaje con emojis, tendencia (📈/📉) y la lista numerada. Varios destinatarios.

🌐 Dashboard web

Tablero público: un equipo a la vez, ficha + 3 gráficos de histórico.

Automatización: una tarea de Windows revisa cada 50 min y refresca el dashboard; solo notifica cuando algo cambia (no satura).
